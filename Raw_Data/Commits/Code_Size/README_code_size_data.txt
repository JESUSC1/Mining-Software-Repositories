Columns/Variables in repoName_code_size CSV Files: 

1. repo_name: The name of the repository.
2. repo_owner: The owner of the repository.
3. commit_id: The unique identifier for the commit.
4. committer_name: The name of the person who committed the code.
5. date: The date and time when the commit was made.
6. code_size_LOC: The number of lines of code added or removed in the commit.
7. repo_size_LOC/file: The size of the repository at the time of the commit, measured in lines of code per file.


Note that the code size and repo size are both measured in lines of code (LOC). The repo size is expressed in lines of code per file (LOC/file) to account for changes in the number of files in the repository over time.
